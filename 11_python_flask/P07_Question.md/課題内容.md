# 勤怠管理システム：要件定義書
1. システムの目的
本システムは、従業員の出退勤時刻および休憩時間を正確に記録し、管理者が月次の勤務集計を効率化することを目的とする。

2. ターゲットユーザー
- 一般従業員:
  - 日々の打刻（出勤・退勤・休憩）と自身の履歴照会を行う。
- 管理者:
  - 全従業員のアカウント管理および、月間の勤務データの閲覧・集計を行う。

3. 機能要件
## フェーズ1[認証・基盤機能]
- ログイン/ログアウト
  - メールアドレスとパスワードによる認証。
- 権限判別
  - 一般ユーザー用と管理者用のメニュー出し分け。

## フェーズ2[打刻機能]
- ボタン打刻:
  - 「出勤」「退勤」「休憩開始」「休憩終了」の4アクション。
- 打刻バリデーション:
  - 出勤していないのに「退勤」は押せない。
  - 休憩中に「退勤」は押せない、などの整合性チェック。

## フェーズ3[履歴・集計機能]
- 個人勤務表:
  - 自身の打刻記録を月ごとにテーブル（表）形式で表示。
- 勤務時間計算:
  - 実働時間 ＝（退勤 － 出勤）－（休憩終了 － 休憩開始）
  - 1日単位、および月間合計の算出。

## フェーズ4[管理者専用機能]
- ユーザー管理（CRUD）:
  - 従業員の新規登録、情報の編集、削除。
- 全社勤務状況参照:
  - 管理者が全従業員のリストから個別の勤務表を閲覧できる機能。

4. 非機能要件
- データ永続化:
  - SQLiteを使用し、Flask-SQLAlchemyで操作すること。
- セキュリティ:
  - パスワードは必ずハッシュ化して保存すること。
- 保守性:
  - Flask-Migrateを使用し、データベースの変更履歴を管理すること。
