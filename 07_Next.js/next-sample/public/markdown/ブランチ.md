# コンフリクト
例）
リモートリポジトリのconfrict1.mdの1行目：abc
ローカルリポジトリのconfrict1.mdの1行目：def

コンフリクトが発生した場合は。左サイドバーにあるソース管理からclickする。
マージエディターをクリックする

左がだれかが作ったもの。　→が自分が作ったもの

左を選択するかor右を選択するか
組み合わせを左ベースにしてマージするか右をベースにしてマージするか。
左をベースに右を無視するか右をベースにして左を無視するか



## ブランチの作成
1. VSCode 左側の ソース管理（Git）パネルを開く

2. 左下の 現在のブランチ名（例：main） をクリック

3. メニューから 「Create Branch...」または「新しいブランチ」 を選択

4. 新しいブランチ名を入力（例：feature/login-ui）

5. Enter → 自動で新しいブランチに切り替わる

<img width=500 src="/授業用/markdown/images/ブランチの作成手順.png"  alt="" >


## 📝 ブランチで作業してコミットする

1. ファイルを変更

2. ソース管理パネルで変更ファイルにチェック

3. 上部の入力欄にコミットメッセージを入力

4. 「✓コミット」ボタンをクリック

5. 「…」メニュー → Push（または自動で push するか確認されます）


## 🔀 GitHub の main へマージする
1. VSCode だけでマージ（ローカルで完結）

2. 左下のブランチ名をクリックして main に切り替え

3. 再度左下のブランチ名（main）をクリック

4. 「Merge Branch...」を選択

5. マージしたいブランチ（例：feature/login-ui）を選ぶ

これでローカルの main にマージされます。

その後：→ 「…」メニュー → Push で GitHub の main に反映される。
