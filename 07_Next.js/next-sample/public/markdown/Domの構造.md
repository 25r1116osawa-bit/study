## HTMLの「設計図」
DOM（ドキュメント・オブジェクト・モデル）を一言で言うと、  
WebブラウザがHTMLを読み込んだときに作成される **「操作可能な設計図」** です。

- **HTML**: Webページの構造を定義したテキストファイル（単なる文字情報）  
- **DOM**: ブラウザがHTMLを解析し、JavaScriptから **プログラムとしてアクセスできる形（オブジェクト）** に変換したもの  

JavaScriptは、このDOMを通じてのみ、  
Webページの文字やデザインを書き換えたり、新しい要素を追加したりできます。

---

## DOMの構造

### ツリー形式（家系図）
DOMの最大の特徴は、HTMLのすべての要素を **ツリー構造（木のような階層構造）** で整理することです。

### ノード（Node）と階層
- ツリー構造の中にある一つ一つの要素を **ノード** と呼びます。

#### ルート（根元）
- ツリーの最上位は **`document` オブジェクト**  
- その下に `<html>` ノードがあります。

#### 親子関係
- HTML要素は、すべて親と子の関係で結ばれています。  
  例：`<body>` の中の `<p>` は `<body>` の子要素  
       `<body>` は `<p>` の親要素

#### 兄弟関係
- 親が同じ要素同士は **兄弟要素** と呼ばれます。

この構造のおかげで、JavaScriptは  
「このボタン（子要素）の親要素を探して、その中にあるリスト（兄弟要素）を操作する」  
といった複雑な操作もできるようになります。



## JavaScriptとDOMの役割

JavaScriptがWebページに「動き」を加えるには、以下の3つの手順でDOMを使います。

1. 要素を選択 (Select)
ツリー構造の中から、操作したいノードを特定します。  
- メソッド例:  

document.querySelector('#id名')

2. 読み書き (Read/Write)

選択したノードの持っている情報（文字やスタイル）を取得したり、変更したりします。

メソッド例:

element.textContent = '新しい文字'

3. イベント処理 (Action)

ノードに「ボタンが押された」「マウスが乗った」などのアクションを監視させ、対応する処理を実行します。

メソッド例:

element.addEventListener('click', function() {
  // クリック時の処理
});


---


1. オブジェクト:"Webページの部品（ボタンや文字、画像など）を表す「箱」

JavaScriptから操作できる形になっている

例：button は button タグのオブジェクト


2. メソッド（関数）	

オブジェクトに対して「何かをさせる命令」

DOMでは、文字を書き換えたり、イベントを登録したりできる

例：addEventListener()、textContent"


3. イベント名	「いつ動くか」をブラウザに知らせるラベル

ブラウザがその名前の出来事を監視して、発生したときに登録した関数を呼ぶ

例：'click'（クリックされたとき）、'mouseover'（マウスが乗ったとき）
[オブジェクト（部品）]  →  [メソッド（命令）]  →  [イベント名（ラベル）で動くタイミングを指定]  →  [関数（お仕事）を実行]

🔍 オブジェクトの { } と function の { } は役割が違う！
▼ オブジェクトの {}

これは データ（プロパティ・メソッド）を入れる箱 を作る。

const person = {
  name: "太郎",
  age: 10
};


person は オブジェクト

{ } の中に プロパティ（name、age） を入れる

▼ 関数の {}

こちらは 処理（実行する命令）をまとめる。

function aisatu() {
  console.log("ごめんなさい");
  console.log("ありがとう");
}


aisatu は 関数

{ } の中は「実行される命令たち（処理）」

🔥 わかりやすいイメージ
● オブジェクト {}

→ 情報を入れる棚

● 関数の {}

→ やる仕事を入れる袋

同じ {} でも用途がまったく違うのです。

🔵 まとめ
種類	{ } の役割	例
オブジェクト	データ（情報）を入れる	{ name:"太郎" }
関数ブロック	実行する命令をまとめる	function(){ ... }
if / for ブロック	条件や繰り返しの中の処理を入れる	if(){ ... }

| 使い方               | `{}` の意味    | 中身       | 例                 |
| ----------------- | ----------- | -------- | ----------------- |
| **オブジェクト**        | データを入れる棚    | 値（プロパティ） | `{ name:"太郎" }`   |
| **関数ブロック**        | 実行する命令の袋    | 命令（処理）   | `function(){...}` |
| **if / for ブロック** | 条件・繰り返しの命令箱 | 命令（処理）   | `if(){...}`       |
