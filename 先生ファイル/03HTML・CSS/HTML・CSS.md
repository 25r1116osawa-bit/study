## 概要

## タグとは

- HTMLの要素を表すための記号
- タグは、開始タグと終了タグで構成される
  - 例：`<p>これは段落です。</p>`
- 一部のタグは終了タグを持たない
  - 例：`<img>`、`<br>`

## HTMLの基本構造

```html
<!doctype html>
<html>
  <head>
    メタデータの部分。ページとしては表示されない
  </head>
  <body>
    コンテンツの部分。ページに表示される
  </body>
</html>
```

## セレクター
- CSSで特定のHTML要素を選択するためのパターン
- 主なセレクターの種類
  - タグセレクター
    - 例：`p { color: blue; }`
  - クラスセレクター
    - 例：`.classname { font-size: 16px; }`
  - IDセレクター
    - 例：`#idname { margin: 10px; }`
  - 属性セレクター
    - 例：`a[href^="https"] { text-decoration: none; }`
  - 擬似クラスセレクター
    - 例：`a:hover { color: red; }`
  - 擬似要素セレクター
    - 例：`p::first-letter { font-size: 200%; }`
  - 子セレクター
    - 例：`div > p { margin: 0; }`
  - 子孫セレクター
    - 例：`div p { line-height: 1.5; }`
  - グループセレクター
    - 例：`h1, h2, h3 { font-family: Arial; }`
  - ユニバーサルセレクター
    - 例：`* { box-sizing: border-box; }`

## 命名規則
- 命名規則とは、プログラムの変数、関数、クラスなどに名前を付けるためのルールです。これにより、コードの可読性、保守性、効率が向上し、チーム内での認識のずれやバグを防ぎます。主な命名規則には、キャメルケース、パスカルケース、スネークケースなどがあり、プロジェクトや言語ごとに共通のルールを設けることが重要です。 
### 命名規則の目的
- 可読性と保守性の向上: コードが何を意味するのか、一目で理解しやすくなります。
- 効率化: 名前を検索したり、コードを理解したりする際の作業効率が上がります。
- チーム開発での一貫性: 複数の開発者がいるチームで、認識のずれや誤認を防ぎます。
- バグの防止: 命名規則が統一されていないと、コードに誤りが生じやすくなります。 
### 主な命名規則
- キャメルケース (camelCase): 最初の単語を小文字で始め、2つ目以降の単語の頭文字を大文字にします。一般的な変数名や関数名に使われます。
  - 例: userId, getFirstName
- パスカルケース (PascalCase): すべての単語の頭文字を大文字にします。主にクラス名やコンストラクタに使われます。
  - 例: UserAccount, ShoppingCart
- スネークケース (snake_case): 単語をすべて小文字にし、アンダースコアで区切ります。
  - 例: user_id, calculate_average
- アッパースネークケース (UPPER_SNAKE_CASE): すべての単語を大文字にし、アンダースコアで区切ります。定数に使われることが多いです。
  - 例: MAX_CONNECTIONS
- ケバブケース (kebab-case): 単語をすべて小文字にし、ハイフンで区切ります。主にURLやCSSクラス名に使われます。
  - 例: user-profile, main-header
### 命名のポイント
- 具体的に: 曖昧な名前を避け、何をしているかがわかるように具体的に命名します。
- 一貫性: 同じ概念には同じ名前を使い、途中で変更しないようにします。
- 適切な長さ: 短すぎず、長すぎない、適切な情報を含む長さを心がけます。
- ドメイン用語の活用: ビジネスロジックに関連する専門用語を使用すると、開発者間のコミュニケーションがスムーズになります。
- 汎用的な名前の回避: data や info のような汎用的な名前は避け、意味が伝わる名前にします。
- 動詞と名詞の使い分け: 関数名には動詞を、変数名やクラス名には名詞を使います。 

## CDNとは
- CDN（Content Delivery Network）は、インターネット上でコンテンツを効率的に配信するための分散型サーバーネットワークです。CDNは、地理的に分散したサーバー群を利用して、ユーザーに最も近いサーバーからコンテンツを提供することで、ウェブサイトの読み込み速度を向上させ、遅延を減少させます。これにより、ユーザーエクスペリエンスが向上し、サーバーの負荷が軽減されます。CDNは、画像、動画、スタイルシート、JavaScriptファイルなどの静的コンテンツの配信に広く利用されています。

### CDNの仕組み
- コンテンツのキャッシュ: ウェブサイトのデータ（画像や動画などの静的コンテンツ）を、各地のCDNサーバーに一時的にコピー（キャッシュ）します。
- 高速な配信: ユーザーがサイトにアクセスすると、オリジナルのサーバーではなく、ユーザーから地理的に最も近いキャッシュサーバーからコンテンツが配信されます。
- アクセス元の近くのサーバー: 物理的な距離だけでなく、ネットワークの経由（ホップ数）が少ないサーバーが自動的に選択されます。 
### CDNのメリット
- 表示速度の向上: ユーザーとサーバーの距離が短縮されるため、Webサイトの表示速度が向上します。
- オリジンサーバーの負荷軽減: アクセスが集中しても、CDNサーバーがリクエストを分散するため、オリジナルのサーバーへの負荷が軽減されます。
- セキュリティの向上: 大量のトラフィックやDDoS攻撃などのサイバー攻撃に対して、負荷分散によってオリジンサーバーを守る効果が期待できます。 
### CDNを利用する場面
- ECサイト: セール時など、アクセスが急増するECサイトでのサーバーダウンを防ぎます。
- 動画配信サービス: 大容量の動画データを迅速に配信するために不可欠です。
- グローバルなWebサイト: 世界中にユーザーがいるサイトで、地域ごとの表示速度を改善します。 

## HTMLフォーム
- ウェブサイトでユーザーから情報を集め、それをサーバー（サイトの裏側）に送るための機能です。
- 例：
  - 会員登録、ログイン、お問い合わせ、注文。

### フォームの基本的な構成要素
- formタグ
  - フォームの枠組み
  - どこにデータを送信するのか？
    - action属性で指定(データの送信先のURL)
  - どうやってデータを送信するのか？
    - method属性で指定(データの送信方法)
      - GET
        - 情報が共有しやすいようにデータはURLに載る。
        - 実は・・Formを使わなくてもこちらのタイプで通信している。
      - POST
        - データを送信する方法で、URLには載らない。
- inputタグ
  - 入力部品
    - サーバーに送信する情報
    - type属性で部品の種類を変更できます。
    - 亜種として
      - textareaタグ
      - selectタグ / optionタグ
- name属性
  - サーバーにデータを送信するとき、そのデータが何のデータなのか識別するための名前
  - すべての入力部品に必須です。

### 主要な入力部品
- type属性を変えることで、さまざまな入力形式を実現します。

| `type`の値 |  表示される部品 | 説明 |
| :--- | :--- | :--- |
| **`text`** | 1行のテキスト入力欄 | 名前や住所など、一般的な文字を入力してもらう場所。 |
| **`password`**| パスワード入力欄 | 入力した文字が「\*」や「•」で隠される入力欄。 |
| `checkbox`| チェックボックス | 複数ある選択肢の中から、いくつでも選べる部品。 |
| `radio` | ラジオボタン | 複数ある選択肢の中から、どれか一つだけを選べる部品。 |
| `submit`| 送信ボタン | フォームの入力を**サーバーへ送る\*\*ためのボタン。 |

## CSSフレームワーク
### フレームワークとは
- 「枠組み」「骨組み」「構造」を意味し、システム開発やビジネスにおける意思決定、問題解決、分析などで活用される、あらかじめ定められた構造や手法のこと

- 目的:
  - アプリケーション開発の効率化と品質向上。﻿
- 役割:
  - よく使う機能（データベースへの登録、検索、画面表示など）が部品として用意されており、これらを土台にして開発を進める。﻿
- メリット:
  - 開発効率の向上:よく使う機能が最初から用意されているため、開発時間が短縮される。﻿
- メンテナンス性の向上:
  - コードの書き方が統一されるため、担当者が変わってもメンテナンスがしやすくなる。﻿
- 代表的な例:
  - Webアプリケーション開発: Ruby on Rails（Ruby）、Django（Python）、Laravel（PHP）、Spring Framework（Java）など。﻿
